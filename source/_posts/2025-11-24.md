Windows 平台调试安卓设备时如果遇到 **ADB** 命令正常

```sh
adb devices
adb reboot bootloader
```

但 **Fastboot** 命令找不到设备

```sh
fastboot devices
```

这时打开设备管理器, 如果在 **其他设备** 选项卡有一个带感叹号的 Android 设备, 这代表缺少 Google USB 驱动

你可以直接 [下载](https://developer.android.google.cn/studio/run/win-usb?hl=zh-cn) 并解压到某个位置

也可以使用 Android Studio 来下载

然后找到设备管理器里有感叹号的设备, 右键, 选择 **更新驱动程序**, 在自己的电脑上选择驱动程序, 然后对于自行解压的, 直接填入包含 `android_winusb.inf` 文件的目录, 对于使用 Android Studio 安装的, 文件位于 `%AndroidSDK%\extras\google\usb_driver\android_winusb.inf`

安装完驱动程序, 此时原安卓设备应该转变成 ADB 设备之类的. 这时进入 Fastboot 模式就可以找到设备了. 可以使用 Fastboot 命令临时刷入内核之类的, 比如 KernelSU

```sh
fastboot boot boot.img # OPPO 系通常是 GZ 压缩格式
```
